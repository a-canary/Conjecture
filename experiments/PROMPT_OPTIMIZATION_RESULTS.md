## Key Findings

### What Works (Winner Patterns)

1. **Interrogative instruction format** - "What is the single most important claim?" > "State your next claim"
2. **Example-based instruction** - Few-shot examples anchor output format effectively
3. **Detailed system prompt** - Provides enough context without being verbose
4. **Natural language output** - Better overall than JSON for small models

### What Hurts (Anti-Patterns)

1. **Reasoning modifier** - CoT/explain-before-answer destroys atomicity (0-30%)
2. **Brevity modifier** - Too aggressive, cuts relevance and confidence
3. **Imperative instruction** - Lower scores across all variants
4. **JSON + no modifier** - Loses relevance without compensating structure

## Optimal Configuration for Small Models

### System Prompt (v10/v14 winner)
```
You are a careful reasoning assistant that builds knowledge through claims.

A claim is a single, atomic statement that captures one piece of reasoning.
Each claim must have: content (what you believe), type (observation/assertion/assumption), and confidence (0.0-1.0).

Output exactly ONE claim per response. Do not combine multiple ideas into one claim.
```

### User Prompt Template (interrogative)
```
What is the single most important claim you can make about: {problem}

Your claim:
```

### Alternative: Example-Based (v14)
```
Problem: What is 2+2?
Claim: "The sum of 2 and 2 equals 4" (assertion, confidence: 1.0)

Problem: {problem}
Claim:
```

---

## Implementation Recommendations

1. **Use `detailed` + `interrogative` or `example`** for optimal single-claim generation
2. **Avoid `imperative` instruction** - consistently underperforms
3. **Avoid `reasoning` modifier** - destroys atomicity on small models (matches prior CoT findings)
4. **Use `natural` output** over JSON for 8B models - better overall scores
5. **If using JSON**, pair with `interrogative` instruction (v19: 0.955)
