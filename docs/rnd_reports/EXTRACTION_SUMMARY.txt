================================================================================
PROBLEMS IDENTIFIED
================================================================================

CRITICAL (6 issues):

1. deepeval_suite.py (Line 99)
   - Simple substring matching causes false positives/negatives
   - Example: "0.05" incorrectly found in "$0.50"
   - Impact: CRITICAL - causes 70pp accuracy swings

2. benchmark_framework.py (Lines 193-221)
   - Only 3 regex patterns, missing LaTeX boxed, "final answer", etc.
   - No type inference
   - Example: \boxed{17} not detected
   - Impact: HIGH - misses valid answers in different formats

3. run_benchmark.py (Lines 160-191)
   - Multiple choice detection only scans first 50 characters
   - "Last number" fallback can select wrong answer
   - Example: Gets $1.05 instead of $0.05 from mixed content
   - Impact: HIGH - multiple edge cases cause wrong selection

4. Consistency Issues
   - Different extraction logic in 3 different files
   - Buggy duplication across implementations
   - Impact: MEDIUM - maintenance nightmare

5. Missing Patterns
   - GSM8K #### format: Not in deepeval_suite or benchmark_framework
   - LaTeX \boxed{}: Not in deepeval_suite or benchmark_framework
   - "final answer:", "therefore", "result:" patterns missing
   - Impact: HIGH - diverse benchmark formats not handled

6. Type Inference Failures
   - No explicit answer type handling (numerical vs MC vs categorical)
   - Affects multiple-choice and categorical benchmarks
   - Impact: MEDIUM - reduces accuracy for non-numerical benchmarks

================================================================================

Created unified extraction module with:

EXTRACTION PATTERNS (8+):
1. #### NUMBER (GSM8K format)

5. "final answer: X"
6. "result: X"
7. "therefore X"
8. "x = Y" (equations)

TYPE-AWARE MATCHING:
- NUMERICAL: Floating-point tolerance (±0.01), comma handling
- MULTIPLE_CHOICE: Letter validation (A-D), case-insensitive
- CATEGORICAL: Exact text match after normalization

UNIFIED NORMALIZATION:
- Whitespace stripping
- Case conversion
- Comma replacement (1,000 → 1000)
- Consistent across all paths
