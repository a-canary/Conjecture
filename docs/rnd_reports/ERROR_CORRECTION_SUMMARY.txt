Key Features:
================================================================================

LIGHTWEIGHT ALTERNATIVE TO FULL RE-GENERATION
  - Initial answer: Model generates with confidence score
  - Confidence check: If below 0.7, trigger correction
  - Error correction: Lightweight prompt-based reconsideration
  - Cost: ~50-500 tokens vs 2x cost for full regeneration

DOMAIN-SPECIFIC ERROR CHECKING
  Mathematical:
    - Calculation review (PEMDAS/BODMAS, arithmetic errors)
    - Logic verification (units, interpretation)
    - Alternative checks (working backwards, estimation)

  Logical:
    - Premise verification (facts, assumptions)
    - Reasoning chain (logical steps, fallacies)
    - Conclusion validity (counterexamples, alternatives)

  Sequential:
    - Step order (correct sequence, prerequisites)
    - Step completeness (no missing steps)
    - Dependency verification (relationships)

  Scientific:
    - Methodology (scientific method application)
    - Evidence interpretation (correlation vs causation)
    - Conclusion support (evidence justification)

  Decomposition:
    - Component identification (all major parts)
    - Component analysis (thorough evaluation)
    - Integration (proper combination)

CONFIDENCE-BASED TRIGGERING
  - Trigger when confidence < 0.7 (configurable)
  - Trigger on uncertainty markers ("maybe", "might", etc.)
  - Trigger on short responses (< 50 chars)
  - Configurable per problem type

Usage Patterns:
================================================================================

PATTERN 1: INLINE ERROR CORRECTION (LIGHTWEIGHT)
  if prompt_system.should_trigger_error_correction(confidence):
      quick_reminder = prompt_system.get_quick_error_correction(problem_type)
      system_prompt += f"\n\nSAFETY CHECK:\n{quick_reminder}"

PATTERN 2: FULL ERROR CORRECTION (TARGETED)
  if confidence < 0.7:
      correction_prompt = prompt_system.get_error_correction_prompt(
          problem, initial_response, problem_type
      )
      corrected = model.generate(correction_prompt)

PATTERN 3: AUTOMATIC WORKFLOW
  1. Get initial answer with confidence
  2. Check if correction needed
  3. Generate domain-specific correction prompt
  4. Get corrected answer
  5. Return best result
